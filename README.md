# TOP by dobryak

Статический сайт для списка фильмов, аниме и сериалов.

## Что уже есть

- список отображается сразу при заходе;
- фильтры: `Недавно / Топ рейтинг`, `Все / Аниме / Фильмы / Сериалы`, `Просмотрено / Запланировано`;
- по нажатию на карточку идет переход в `ya.ru` по названию;
- добавление нового пункта через кнопку `+`;
- рейтинг и категории задаются в форме;
- данные сохраняются в `localStorage`, то есть не пропадают после перезагрузки браузера;
- поиск 2-3 картинок идет по Яндекс.Картинкам через публичные CORS-прокси.

## Как запустить локально

### Вариант 1

Открой `index.html` двойным кликом.


## Как бесплатно выложить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузи в него файлы из этой папки.
3. Открой `Settings -> Pages`.
4. В `Build and deployment` выбери:
   - `Source: Deploy from a branch`
   - `Branch: main`
   - `/ (root)`
5. Сохрани.
6. Через 1-3 минуты сайт появится по адресу вида `https://твой-логин.github.io/имя-репозитория/`.

## Как подключить любой свой домен

Для `subdomain.example.com`:

1. В `GitHub -> Settings -> Pages -> Custom domain` впиши `subdomain.example.com`.
2. У регистратора домена создай `CNAME`:

```txt
subdomain.example.com -> твой-логин.github.io
```

Для корневого домена `example.com`:

1. В `GitHub -> Settings -> Pages -> Custom domain` впиши `example.com`.
2. У регистратора создай записи `A` на GitHub Pages:

```txt
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

3. Желательно дополнительно сделать `www` через `CNAME` на `твой-логин.github.io`.

Официальная документация GitHub по доменам:
https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site

## Ограничение по картинкам

Автопоиск картинок работает без собственного сервера, поэтому использует публичные CORS-прокси. Обычно этого достаточно, но иногда прокси или Яндекс могут временно не ответить. В этом случае в форме есть кнопка открыть Яндекс.Картинки вручную.
